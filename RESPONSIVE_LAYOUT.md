# 后台管理页面响应式布局改造说明

## 概述

本次改造将后台管理页面升级为响应式布局，完美兼容移动端设备（手机、平板）和桌面端。

## 主要改动

### 1. 主布局 (App.tsx)

**核心特性：**
- ✅ 响应式侧边栏：在移动端（< 768px）自动折叠
- ✅ 汉堡菜单按钮：方便移动端用户展开/收起侧边栏
- ✅ 动态 padding：移动端使用更小的间距
- ✅ 固定定位侧边栏：移动端侧边栏固定在左侧，不占用文档流

**断点说明：**
- `xs`: < 576px（手机）
- `sm`: ≥ 576px（大手机）
- `md`: ≥ 768px（平板，侧边栏断点）
- `lg`: ≥ 992px（小桌面）
- `xl`: ≥ 1200px（桌面）

### 2. 支付订单页面 (PayOrderPage.tsx)

**表单响应式：**
- 桌面端：行内表单 (inline)
- 移动端：垂直表单 (vertical)
- Grid 布局：使用 Col 的 xs/sm/md/lg 属性

**表格优化：**
- 使用 `responsive` 属性隐藏次要列
- 移动端使用 `small` 尺寸
- 自适应滚动：`scroll={{ x: 800 }}`
- Avatar 大小自适应

### 3. 用户管理页面 (UserPage.tsx)

**统计卡片：**
- 响应式栅格：`xs={24} sm={12} md={12} lg={6}`
- 字体大小自适应
- 图表高度动态调整（移动端 250px，桌面端 400px）

**表格列优化：**
- ID 列：仅在 md+ 显示
- 微信ID：仅在 lg+ 显示
- 时间列：lg+ 显示完整时间，否则只显示日期

### 4. 系统配置页面 (SystemConfig.tsx)

**表格优化：**
- ID、版本列响应式隐藏
- Switch 按钮尺寸自适应
- 代码字体大小响应式
- Modal 宽度：移动端 90%，桌面端 520px

### 5. 全局样式 (index.css)

**移动端优化：**
- 表格横向滚动
- 卡片间距调整
- 按钮最小高度 36px（方便触控）
- Modal 宽度限制
- 分页器居中

**小屏幕优化（< 576px）：**
- 紧凑的表格样式
- 更小的字体
- 优化的表单布局

**其他：**
- 自定义滚动条样式
- ECharts 图表响应式高度
- 触摸滚动优化

## 使用的技术

1. **Ant Design Grid System**
   - `Grid.useBreakpoint()` Hook
   - `Row` 和 `Col` 响应式栅格
   - `responsive` 属性（Table 列）

2. **响应式属性**
   - `size`：组件尺寸（small/middle/default）
   - `layout`：表单布局（inline/vertical）
   - `scroll`：表格滚动配置

3. **CSS Media Queries**
   - 移动端：`@media (max-width: 768px)`
   - 小屏幕：`@media (max-width: 576px)`
   - 平板：`@media (min-width: 768px) and (max-width: 992px)`
